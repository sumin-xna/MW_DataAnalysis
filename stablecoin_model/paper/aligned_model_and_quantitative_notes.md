# Code-aligned model and quantitative notes

This note aligns the paper draft to the implemented file `code/hedge_run_structural_welfare.py`.

## 1. State variable

The model is solved on a grid of overvaluation states:

- theta in [0, 1.4]

This is an overvaluation-only prototype. The model is therefore not written as a full global-games integration over heterogeneous private signals in the current code. Instead, it is a reduced-form deterministic state-space prototype that keeps the two key economic margins: hedge demand and run demand.

## 2. Information channel

The code defines public-information quality as a function of stablecoin demand.

Precision components:
- prec_theta = 1 / sigma_theta^2
- prec_x = 1 / sigma_x^2
- sigma_s^2 = sigma_s_bar^2 / (1 + rho * Qs)
- prec_s = 1 / sigma_s^2

Public-signal weight:
- omega_s = prec_s / (prec_theta + prec_x + prec_s)

Total posterior precision:
- precision = prec_theta + prec_x + prec_s

Economic interpretation:
- more stablecoin usage makes the public signal more precise,
- more precise public information raises the coordination component of FX demand.

## 3. Instrument prices and FX access costs

The code uses square-root price schedules:

- p_c = a_c + b_c * sqrt(Qc)
- p_s = a_s + b_s * sqrt(Qs)

Instrument-specific total costs:
- cash cost = p_c + kappa_c
- stablecoin cost = p_s + kappa_s

Stablecoin share rule:
- share_s = logistic( nu_share * [ (p_c + kappa_c) - (p_s + kappa_s) ] + 1.2 * omega_s )
- share_c = 1 - share_s

Effective FX acquisition cost:
- eff_cost = share_c * (p_c + kappa_c) + share_s * (p_s + kappa_s)

Economic interpretation:
- stablecoins gain share when they are cheaper than cash,
- stablecoins also gain share when the public signal is better.

## 4. FX demand decomposition

The code implements total foreign-currency demand as:

- Qfx = q_h + q_r

### Hedge demand

- q_h = max(0, chi_h * theta - eta_h * eff_cost)

Interpretation:
- higher overvaluation raises the desire to hold FX to preserve purchasing power,
- higher acquisition cost reduces hedge demand.

### Run demand

- q_r = gamma_r * (1 + psi_info * omega_s) * logistic( (pi - pi0_r) / tau_r )

Interpretation:
- run demand rises with crisis beliefs,
- run demand is stronger when the public signal is more informative,
- stablecoins therefore amplify strategic demand through omega_s.

## 5. Allocation across instruments

Given total FX demand:

- Qs = share_s * Qfx
- Qc = share_c * Qfx

This means stablecoins affect equilibrium both directly through the cost margin and indirectly through the information margin.

## 6. Crisis probability

The code defines crisis probability as a smooth logistic function:

- threshold = alpha - delta * theta
- pi = logistic( (Qfx - threshold) / tau_crisis )

Interpretation:
- overvaluation weakens effective defense capacity,
- higher aggregate FX demand increases crisis probability,
- the crisis mapping is smooth rather than discontinuous.

This is economically useful because it avoids knife-edge collapse behavior while preserving the idea that crises occur when aggregate demand outstrips usable defense capacity.

## 7. Welfare block

The current code defines welfare as expected log utility over a normal state and a crisis state.

Normal-state consumption:
- c_normal = 1 + zeta * log(1 + q_h + 0.25 * q_r) - eff_cost * Qfx

Crisis-state loss:
- crisis_loss = lam_loss * theta * (1 + xi_coord * omega_s) * (1 + q_r)

Crisis-state consumption:
- c_crisis = max(c_normal - crisis_loss, 1e-5)

Expected welfare:
- W = (1 - pi) * log(c_normal) + pi * log(c_crisis)

Economic interpretation:
- the normal state values access to tradables and FX smoothing,
- the crisis state penalizes overvaluation, strategic run intensity, and coordination,
- stablecoins improve welfare through lower access costs but can reduce welfare by raising coordination-sensitive crisis losses.

## 8. Why this version makes economic sense

This code-aligned model is economically coherent for four reasons.

First, it separates two motives for FX demand that should not be compressed into a single reduced-form choice margin:
- hedging against gradual misalignment,
- running in response to crisis fears.

Second, it gives stablecoins two conceptually distinct roles:
- a lower-cost access technology,
- an endogenous public signal.

Third, it makes crisis risk depend on aggregate FX pressure relative to a declining defense threshold.

Fourth, it defines welfare over normal and crisis states instead of inheriting welfare mechanically from a static asset-choice utility index.

## 9. What should be changed in the paper

The paper should present this as a reduced-form structural prototype rather than as a full heterogeneous-agent Bayesian game. In particular:

1. Remove language suggesting that the current quantitative code explicitly integrates over household-level private signals.
2. Present omega_s as the reduced-form public-information object generated by stablecoin usage.
3. Present q_h and q_r as the two central behavioral margins.
4. Present welfare as a state-contingent consumption object rather than an abstract asset utility.

## 10. Main propositions consistent with the code

The paper can safely claim the following code-consistent results.

### Proposition 1. Monotone FX demand
Total FX demand is increasing in theta because hedge demand is increasing in theta and run demand is weakly increasing in crisis beliefs.

### Proposition 2. Monotone stablecoin demand
Stablecoin demand is increasing in theta because total FX demand rises and stablecoins remain the lower-cost instrument over the relevant region.

### Proposition 3. Endogenous public information
Stablecoin usage increases public-signal precision and raises omega_s.

### Proposition 4. Crisis amplification
Stablecoins increase crisis probability by increasing both FX access and coordination.

### Proposition 5. Welfare reversal
Stablecoins improve welfare in moderate-theta states but reduce welfare in high-theta states when crisis-sensitive disruption losses dominate the access benefit.

## 11. Recommended paper language

The cleanest way to describe the model is:

> We develop a reduced-form structural model of parallel FX demand in which households demand foreign currency for both hedging and strategic reasons. Stablecoins reduce the cost of acquiring foreign currency and endogenously increase the precision of public information. These two channels jointly determine FX demand, crisis risk, and welfare.

This language is fully aligned with the implemented code and avoids overstating what the current numerical prototype does.
